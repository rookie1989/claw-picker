## 二、核心概念（4:30 - 6:30）

**[画面]**：架构图动画

**[旁白]**：

> 在 OpenClaw 的多 Agent 架构中，有三个核心角色：  
>  
> **第一个：Orchestrator（协调者）**  
> 负责接收任务、拆分子任务、分配给专家、汇总结果。  
> 就像一个项目经理。  
>  
> **第二个：Worker Agent（工人 Agent）**  
> 每个 Worker 专注于一个领域。  
> 研究员、写作者、翻译、代码审查……每种都是独立的 Agent。  
>  
> **第三个：Critic Agent（评审者）**  
> 对其他 Agent 的输出进行质量把关。  
> 错了就打回去重做。

**[画面]**：代码示意图

> 在 OpenClaw 中，每个 Agent 对应一个**渠道**（Channel）。  
> 不同渠道可以用不同的模型、不同的系统提示词、不同的温度参数。

## 三、配置多渠道（6:30 - 9:00）

**[画面]**：屏幕录制 - 打开 OpenClaw 配置文件

**[旁白]**：

> 现在来配置我们的三个专家渠道。打开配置文件：

```bash
nano ~/.openclaw/config.yaml
```

**[画面]**：输入配置代码

> 第一个渠道：研究员。  
> 模型用 gpt-4o-mini，温度调到 0.1，让它尽量准确、不发挥。

```yaml
channels:
  - name: researcher
    model: gpt-4o-mini
    temperature: 0.1
    system_prompt: |
      你是一个专业的信息研究员。
      职责：准确收集和整理信息，标注不确定内容。
      保持客观，不添加个人观点。
```

> 第二个渠道：写作者。  
> 模型用 gpt-4o，温度调到 0.7，给它更多创意空间。

```yaml
  - name: writer
    model: gpt-4o
    temperature: 0.7
    system_prompt: |
      你是一个资深内容创作者。
      基于给定素材创作高质量文章，确保逻辑流畅、语言生动。
```

> 第三个渠道：评审者。  
> 温度最低，因为我们希望它严格、客观。

```yaml
  - name: critic
    model: gpt-4o
    temperature: 0.1
    system_prompt: |
      你是一个严格的内容审查专家。
      检查逻辑错误和事实不准确。
      输出格式：{"score": 1-10, "issues": [], "suggestions": []}
```

**[画面]**：保存文件，重启服务

```bash
openclaw restart
openclaw channels list  # 验证三个渠道都在线
```

## 四、三种编排模式（9:00 - 12:30）

**[画面]**：三种模式的对比动画

**[旁白]**：

### 模式一：管道（Pipeline）

> 最简单的模式：A 处理完传给 B，B 处理完传给 C。  
> 适合有明确先后顺序的任务。  

```javascript
// 研究 → 写作 → 审查，依次进行
async function pipeline(topic) {
  const research = await callAgent('researcher', `研究「${topic}」的核心信息`);
  const draft    = await callAgent('writer',     `基于以下素材写文章：\n${research}`);
  const review   = await callAgent('critic',     `审查以下文章：\n${draft}`);
  return { research, draft, review };
}
```

**[画面]**：动画展示数据流向

> 缺点是速度慢——必须等前一个完成才能开始下一个。

### 模式二：并行（Fan-out）

> 多个 Agent 同时工作，最后汇总结果。  
> 适合相互独立的子任务。  

```javascript
// 安全审查、性能审查、风格审查，同时进行
const [security, perf, style] = await Promise.all([
  callAgent('security-reviewer', code),
  callAgent('perf-reviewer', code),
  callAgent('style-reviewer', code)
]);
```

> 时间缩短到原来的三分之一！

### 模式三：动态路由（Router）

> 先让一个路由 Agent 判断任务类型，再分发给对应专家。  
> 适合客服、问答等需要分类处理的场景。  

```javascript
const category = await callAgent('router', `分类以下问题：${question}`);
const answer   = await callAgent(category,  question);
```

## 五、实战：内容创作流水线（12:30 - 16:30）

**[画面]**：完整代码演示

**[旁白]**：

> 好，现在来做一个真实的项目：**自动化科技文章生成系统**。  
> 我们只需要给定一个话题，系统自动完成：研究 → 写作 → 审查 → 优化。  

**[画面]**：创建 `content-pipeline.js` 文件

> 先安装依赖：

```bash
npm install axios
```

> 然后写我们的核心逻辑：

**[画面]**：逐段展示代码，解释每个部分

```javascript
const axios = require('axios');

async function callAgent(channel, prompt) {
  const res = await axios.post('http://localhost:3000/api/v1/chat', {
    channel,
    messages: [{ role: 'user', content: prompt }]
  }, {
    headers: { Authorization: `Bearer ${process.env.OPENCLAW_API_KEY}` }
  });
  return res.data.choices[0].message.content;
}

async function createArticle(topic) {
  // Step 1: 研究
  console.log('📚 研究员正在收集资料...');
  const research = await callAgent('researcher',
    `收集关于「${topic}」的核心信息：背景、现状、数据、常见误解`
  );

  // Step 2: 写作
  console.log('✍️  写作者正在创作...');
  const draft = await callAgent('writer',
    `基于以下研究素材，写一篇800字科普文章：\n\n${research}`
  );

  // Step 3: 审查
  console.log('🔬 评审者正在审查...');
  const reviewJson = await callAgent('critic', `审查这篇文章：\n\n${draft}`);
  const review = JSON.parse(reviewJson);

  // Step 4: 如分数低于8分，优化
  let final = draft;
  if (review.score < 8) {
    console.log(`⚡ 得分 ${review.score}/10，正在优化...`);
    final = await callAgent('writer',
      `根据反馈优化文章：\n问题：${review.issues.join('；')}\n\n原文：\n${draft}`
    );
  }

  return { topic, research, draft, review, final };
}
```

**[画面]**：运行程序，实时展示输出

```bash
OPENCLAW_API_KEY=your-key node content-pipeline.js
```

> 看，三个 Agent 依次工作，最终生成了一篇经过优化的文章！

## 六、进阶：代码审查系统（16:30 - 19:00）

**[画面]**：新的 demo 演示

**[旁白]**：

> 再来一个更酷的例子：**三个 Agent 并行审查代码**，比单个 Agent 更全面。  

**[画面]**：展示问题代码

> 这段代码有什么问题？让我们的三个专家同时来找找：

```javascript
// 三个维度并行审查
const [security, performance, style] = await Promise.all([
  callAgent('security-reviewer',     `安全审查：\n${code}`),
  callAgent('performance-reviewer',  `性能审查：\n${code}`),
  callAgent('style-reviewer',        `规范审查：\n${code}`)
]);

// 汇总报告
const report = await callAgent('orchestrator',
  `综合以下三份审查报告，生成最终审查报告：
   安全：${security}
   性能：${performance}
   规范：${style}`
);
```

**[画面]**：展示输出的审查报告

> 看到了吗：安全 Agent 发现了 SQL 注入漏洞，性能 Agent 发现了 N+1 查询问题，
> 规范 Agent 发现了缺少注释。三个角度，一次覆盖！
