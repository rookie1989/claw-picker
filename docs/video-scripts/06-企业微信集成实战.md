## 📦 第一部分：企业微信两种集成方式（0:45 - 2:30）

**画面：** 对比图表

**旁白：**

> 在开始之前，先说一下企业微信有两种接入方式，我们都会讲到。

**屏幕展示表格：**

| 方式 | 特点 | 适用场景 |
|------|------|---------|
| 群机器人 Webhook | 配置最简单，5 分钟搞定；但只能主动推送，不能收用户消息 | 告警通知、定时播报 |
| 企业自建应用 | 完整双向对话；需要配置回调服务器 | AI 助手、工单系统 |

**旁白：**

> 简单来说：群机器人只能「说话」，自建应用可以「对话」。
>
> 我们先从简单的群机器人开始，5 分钟搞定，然后再进阶到自建应用。

## 🤖 第二部分：群机器人配置（2:30 - 5:30）

**画面：** 录屏演示

### Step 1：创建群机器人

**旁白：**

> 打开企业微信，进入任意群聊，点击右上角「···」→「添加机器人」→「新建机器人」。
>
> 给机器人起个名字，比如「OpenClaw 助手」，点击确定，复制这个 Webhook 地址。

**画面：** 放大显示 Webhook URL

### Step 2：OpenClaw 配置

**旁白：** 切换到编辑器

> 打开 `openclaw.config.yaml`，添加企业微信推送配置：

**画面：** 输入代码

```yaml
notifications:
  wecom:
    webhook: https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=你的KEY
    enabled: true
```

**旁白：**

> 保存，重启 OpenClaw。现在可以测试一下，在终端执行：

```bash
openclaw notify --channel wecom --message "Hello from OpenClaw! 🦞"
```

**画面：** 企业微信群里收到消息

**旁白：**

> 看！消息发出来了！群机器人接入完成，总共用了不到 3 分钟。

## ⚡ 第三部分：自建应用 - 双向对话（5:30 - 11:00）

**画面：** 企业微信后台管理页面

### Step 1：创建企业应用

**旁白：**

> 现在来做更强大的——双向对话。需要在企业微信后台创建自建应用。

**操作步骤演示：**

> 打开 [work.weixin.qq.com](https://work.weixin.qq.com) → 应用管理 → 创建应用 → 填写名称「OpenClaw 智能助手」→ 选择可见范围 → 创建。
>
> 复制三个关键信息：企业 ID（CorpID）、AgentID、Secret。

**画面：** 放大显示三个配置项

### Step 2：配置回调服务器

**旁白：**

> 这是最关键的一步。企业微信要把用户消息发到你的服务器，所以我们需要一个公网 HTTPS 地址。
>
> 本地测试我用 ngrok：

```bash
ngrok http 3000
# 复制 https://xxx.ngrok.io 这个地址
```

**旁白：**

> 在企业微信后台，应用详情 → 接收消息 → 设置 API 接收，填入：
> - URL：`https://xxx.ngrok.io/wecom/callback`
> - Token 和 EncodingAESKey：点「随机」生成，复制保存

### Step 3：代码实现

**画面：** VS Code 展示代码

**旁白：**

> OpenClaw 已经内置了企业微信适配器，我们只需要配置：

```yaml
# openclaw.config.yaml
channels:
  wecom-app:
    type: wecom_app
    corp_id: ${WECOM_CORP_ID}
    agent_id: ${WECOM_AGENT_ID}
    secret: ${WECOM_SECRET}
    token: ${WECOM_TOKEN}
    encoding_aes_key: ${WECOM_AES_KEY}
    ai_router:
      default: gpt-4o
```

**旁白：**

> 环境变量配置好后，启动服务：

```bash
npm start
```

> 回到企业微信后台，点「保存」，服务器验证通过！

**画面：** 手机打开企业微信，找到刚才的应用

**旁白：**

> 现在在应用对话框里发一条消息试试……

**画面：** 手机发送「帮我写一封请假邮件，明天请一天病假」，AI 回复完整邮件

**旁白：**

> 完美！AI 助手在企业微信里双向对话成功！

## 🏗️ 第四部分：实战案例 - 智能工单助手（11:00 - 16:00）

**画面：** 动画展示工作流

**旁白：**

> 好，群机器人和双向对话都搞定了。现在来做一个实际的业务场景——智能工单助手。

**演示流程：**

> 需求很简单：员工在企业微信里描述问题，AI 自动生成工单，分配给对应同事，并发群消息通知。

### 工作流设计

```
员工发消息："网络连不上，服务器宕机了"
        ↓
OpenClaw 接收
        ↓
AI 分类：类型=基础设施，优先级=P1（紧急）
        ↓
自动创建工单（写入数据库）
        ↓
通知运维群 @负责人：⚠️ P1 工单 #1024，请立即处理
        ↓
回复员工：您的工单已创建，工单号 #1024，运维同学会在 15 分钟内响应
```

**画面：** 展示代码

```javascript
// plugins/smart-ticket.js
module.exports = {
  name: 'smart-ticket',
  
  async onMessage({ userId, content, reply, notify }) {
    // AI 分类工单
    const ticket = await this.claw.structured({
      prompt: `分析以下问题，返回 JSON：
问题：${content}
返回格式：{"type": "类型", "priority": "P0/P1/P2/P3", "summary": "简短描述", "assignTo": "负责人"}`,
      schema: TicketSchema
    });
    
    // 写入数据库
    const ticketId = await db.createTicket({ userId, ...ticket, content });
    
    // 通知运维群
    await notify.wecom({
      webhook: process.env.OPS_GROUP_WEBHOOK,
      content: `⚠️ 新工单 #${ticketId} [${ticket.priority}]\n` +
               `类型：${ticket.type}\n` +
               `描述：${ticket.summary}\n` +
               `@${ticket.assignTo} 请处理`
    });
    
    // 回复员工
    await reply(`✅ 已为您创建工单 **#${ticketId}**\n` +
                `优先级：${ticket.priority}\n` +
                `预计响应时间：${getResponseTime(ticket.priority)}`);
  }
};
```

**画面：** 完整演示一遍流程

**旁白：**

> 看！员工发消息，AI 自动处理，运维群收到通知，员工拿到工单号，整个过程不到 2 秒！

## 📚 第五部分：常见问题 & 最佳实践（16:00 - 17:30）

**画面：** 幻灯片

**旁白：**

> 几个重要提示：

**1. 5 秒超时问题**

> 企业微信要求 5 秒内回复。如果 AI 处理时间较长，先回复「正在处理中...」，再通过客服消息接口发送结果。

**2. 消息去重**

> 网络问题可能导致重复消息，OpenClaw 内置了消息 ID 去重，默认开启，不用担心。

**3. 安全性**

> Token 和 AES Key 务必存在环境变量，不要硬编码在代码里！

**4. 测试环境**

> 建议单独建一个测试企业微信应用，和生产环境隔离。
