## 🤖 方案一：群机器人（快速上手）

### 步骤 1：创建群机器人

1. 打开钉钉客户端 → 目标群聊
2. 点击右上角「群设置」→「机器人」→「添加机器人」
3. 选择「自定义」机器人
4. 配置安全设置（三选一）：
   - **加签**（推荐）：复制 Secret 密钥
   - **关键词**：设置触发关键词（如 `AI`、`助手`）
   - **IP 白名单**：填写服务器 IP

5. 复制 **Webhook URL**：
```
https://oapi.dingtalk.com/robot/send?access_token=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### 步骤 2：配置加签验证

```json
{
  "channels": {
    "dingtalk-webhook": {
      "type": "dingtalk-webhook",
      "enabled": true,
      "webhook_url": "https://oapi.dingtalk.com/robot/send?access_token=YOUR_TOKEN",
      "secret": "SECxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx",
      "name": "钉钉群机器人",
      "options": {
        "sign_enabled": true,
        "at_all": false
      }
    }
  }
}
```

### 步骤 3：发送消息测试

```bash
# 发送文本消息
openclaw send --channel dingtalk-webhook "🦞 OpenClaw 已连接钉钉！"

# 发送 Markdown 消息
openclaw send --channel dingtalk-webhook --type markdown "## 测试 \n\n> 来自 OpenClaw"
```

## 🔌 方案二：企业内部机器人（双向交互）

### 步骤 1：创建钉钉应用

1. 访问[钉钉开放平台](https://open.dingtalk.com/)
2. 「应用开发」→「企业内部应用」→「创建应用」
3. 选择「机器人」类型
4. 填写基本信息：
   - 应用名称：`OpenClaw AI 助手`
   - 应用描述：`基于 OpenClaw 的智能 AI 机器人`
   - 应用图标：上传 Logo

### 步骤 2：配置机器人能力

在应用页面 → 「能力」→「机器人」→「添加能力」

配置消息接收地址：
```
https://your-server.com/webhook/dingtalk
```

### 步骤 3：获取应用凭证

在「基础信息」页面获取：
```
AppKey:    dingxxxxxxxxxxxxxxxxxx
AppSecret: xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### 步骤 4：完整配置

```json
{
  "channels": {
    "dingtalk": {
      "type": "dingtalk",
      "enabled": true,
      "app_key": "dingxxxxxxxxxxxxxxxxxx",
      "app_secret": "your_app_secret_here",
      "robot_code": "dingxxxxxxxxxxxxxxxx",
      "name": "钉钉 AI 助手",
      "options": {
        "auto_reply": true,
        "stream_mode": true,
        "welcome_message": "👋 你好！我是 OpenClaw AI 助手\n\n你可以直接向我提问，支持：\n• 💬 自然语言对话\n• 🔍 信息查询\n• 📝 内容生成\n• 🔧 工具使用"
      }
    }
  }
}
```

> 💡 **推荐使用 Stream 模式**：钉钉机器人支持长连接模式，无需公网 IP，本地即可开发测试！

### 步骤 5：启用 Stream 模式（推荐）

Stream 模式是钉钉特有的功能，通过长连接接收消息，无需公网服务器：

```bash
# 启动 OpenClaw（自动建立 Stream 连接）
openclaw start --channel dingtalk

# 查看连接状态
# [DingtalkChannel] Stream 连接已建立 ✅
# [DingtalkChannel] 等待消息中...
```

## 📨 主动推送工作通知

### 推送给指定用户

```javascript
const dingtalk = openclaw.getChannel('dingtalk');

// 推送工作通知（出现在"工作通知"tab）
await dingtalk.sendWorkNotification({
  userIdList: ['user001', 'user002'],
  msg: {
    msgtype: 'markdown',
    markdown: {
      title: '⚠️ 紧急任务提醒',
      text: `## 你有一个紧急任务\n\n**任务：** 生产环境异常处理\n**发现时间：** ${new Date().toLocaleString('zh-CN')}\n**处理人：** @user001\n\n请立即查看并处理！`
    }
  }
});
```

### 定时推送示例

```javascript
// cron/dingtalk-standup.js
const cron = require('node-cron');

module.exports = {
  name: 'dingtalk-standup',
  
  setup: (openclaw) => {
    // 每天 9:30 发送站会提醒
    cron.schedule('30 9 * * 1-5', async () => {
      const team = await getTeamMembers();
      const today = new Date().toLocaleDateString('zh-CN');
      
      await openclaw.getChannel('dingtalk').sendGroupMessage({
        robotCode: process.env.DINGTALK_ROBOT_CODE,
        userIds: team.map(m => m.userId),
        msgKey: 'sampleMarkdown',
        msgParam: JSON.stringify({
          title: `📅 ${today} 站会提醒`,
          text: `## 🌅 早上好，团队！\n\n今天是 **${today}（周${getWeekDay()}）**\n\n**站会议题：**\n1. 昨日完成了什么？\n2. 今日计划做什么？\n3. 是否有任何阻碍？\n\n请在群里发送你的站会更新 👆\n\n或者直接问我：\`/standup 昨天完成了xxx，今天计划xxx\``
        })
      });
    });
  }
};

function getWeekDay() {
  const days = ['日', '一', '二', '三', '四', '五', '六'];
  return days[new Date().getDay()];
}
```

## 🔐 签名验证

所有来自钉钉的请求都需要验证签名，OpenClaw 自动处理：

```javascript
// OpenClaw 内部签名验证逻辑（参考）
const crypto = require('crypto');

function verifySignature(timestamp, secret, signature) {
  const stringToSign = `${timestamp}\n${secret}`;
  const hmac = crypto.createHmac('sha256', secret);
  hmac.update(stringToSign, 'utf8');
  const expectedSign = hmac.digest('base64');
  
  return expectedSign === signature;
}

// 验证消息来源
openclaw.use('dingtalk:middleware', (req, res, next) => {
  const { timestamp, sign } = req.headers;
  
  if (!verifySignature(timestamp, process.env.DINGTALK_SECRET, sign)) {
    return res.status(401).json({ error: 'Invalid signature' });
  }
  
  next();
});
```

## 🛠️ 常见问题

### Q1: 消息发送失败（错误码 310000）

**原因**：签名计算错误  
**解决**：检查 Secret 是否正确，时间戳单位是毫秒

```javascript
// 正确的签名生成方式
const timestamp = Date.now().toString();  // 毫秒时间戳
const sign = generateSign(timestamp, secret);
```

### Q2: 机器人无响应

**排查步骤**：
1. `openclaw status` 确认服务运行
2. 检查消息接收地址配置
3. 如用 Stream 模式，检查连接状态
4. 查看日志：`openclaw logs --channel dingtalk --tail 50`

### Q3: 只能发送文本，无法发 Markdown

**原因**：群机器人需要消息 title 字段  
**解决**：
```javascript
// Markdown 消息必须包含 title
{
  msgtype: 'markdown',
  markdown: {
    title: '消息标题（必填！）',  // ← 不能省略
    text: '## 正文内容'
  }
}
```

### Q4: Stream 模式连接断开

**解决**：OpenClaw 支持自动重连，也可手动：
```bash
openclaw restart --channel dingtalk
```
