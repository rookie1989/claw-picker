# 🏢 企业微信集成指南

**难度**: ⭐⭐⭐  
**预计时间**: 45-60 分钟  
**前置条件**: 已安装 OpenClaw、拥有企业微信管理员权限

---

## 🎯 本文目标

完成本指南后，你将能够：

- ✅ 将 OpenClaw 接入企业微信群机器人
- ✅ 在企业微信中与 AI 助手对话
- ✅ 配置消息过滤与路由规则
- ✅ 实现企业微信 Webhook 消息推送
- ✅ 接入企业微信应用消息（主动推送）

### 1. 企业微信账号要求

| 角色 | 权限 | 说明 |
|------|------|------|
| 管理员 | 应用管理权限 | 创建自建应用、配置 Webhook |

## 🤖 方案一：群机器人（最简单）

### 适用场景

- 团队群组通知
- 无需双向交互的场景
- 快速上手验证

### 步骤 1：创建群机器人

1. 打开企业微信客户端
2. 进入目标群聊
3. 点击右上角「...」→「群机器人」→「添加机器人」
4. 选择「新创建一个机器人」
5. 填写机器人名称（如 `OpenClaw Bot`）
6. 点击「添加」，复制 **Webhook URL**

```
https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
```

### 步骤 2：配置 OpenClaw

编辑 `~/.openclaw/config.json`：

```json
{
  "channels": {
    "wecom-webhook": {
      "type": "wecom-webhook",
      "enabled": true,
      "webhook_url": "https://qyapi.weixin.qq.com/cgi-bin/webhook/send?key=YOUR_KEY",
      "name": "企业微信群机器人"
    }
  }
}
```

### 步骤 3：测试推送

```bash
# 测试发送文本消息
openclaw send --channel wecom-webhook "Hello from OpenClaw! 🦞"

# 测试发送 Markdown
openclaw send --channel wecom-webhook --type markdown "## 测试消息\n\n> 来自 OpenClaw"
```

### 消息格式支持

企业微信群机器人支持以下消息类型：

```javascript
// 文本消息
{
  "msgtype": "text",
  "text": {
    "content": "你好，这是一条文本消息",
    "mentioned_list": ["@all"]  // 可@全员或指定成员
  }
}

// Markdown 消息（推荐）
{
  "msgtype": "markdown",
  "markdown": {
    "content": "# 标题\n> 引用内容\n\n**加粗文字**\n\n[链接](https://example.com)"
  }
}

// 图文消息
{
  "msgtype": "news",
  "news": {
    "articles": [{
      "title": "文章标题",
      "description": "文章描述",
      "url": "https://example.com",
      "picurl": "https://example.com/image.png"
    }]
  }
}
```

## 🔌 方案二：自建应用（完整双向交互）

### 适用场景

- 需要双向对话（用户发消息，AI 回复）
- 需要主动推送消息给指定用户
- 需要精细权限控制

### 步骤 1：创建自建应用

1. 登录[管理后台](https://work.weixin.qq.com/wework_admin/frame#/apps/createApiApp)
2. 点击「创建应用」
3. 填写应用信息：
   - 应用 Logo（上传图标）
   - 应用名称：`OpenClaw AI 助手`
   - 应用介绍：`AI 智能助手，随时随地与 AI 对话`
4. 设置可见范围（建议先选「指定成员」测试）
5. 点击「创建应用」

记录以下信息：
```
AgentId: 1000xxx
Secret:  xxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### 步骤 2：配置消息接收

在应用详情页 → 「接收消息」→「设置 API 接收」：

| 字段 | 说明 | 示例 |
|------|------|------|
| URL | OpenClaw 的 Webhook 端点 | `https://your-server.com/webhook/wecom` |
| Token | 自定义随机字符串 | `MyOpenClawToken123` |
| EncodingAESKey | 43位随机字符串 | 点击「随机生成」 |

> ⚠️ **注意**：URL 必须是公网可访问的 HTTPS 地址，本地开发可使用 [ngrok](https://ngrok.com/) 进行内网穿透。

### 步骤 3：内网穿透（本地开发）

```bash
# 安装 ngrok
brew install ngrok

# 启动 OpenClaw（默认 3000 端口）
openclaw start

# 另开终端，启动 ngrok
ngrok http 3000

# 复制 ngrok 提供的 HTTPS 地址
# 例如: https://abc123.ngrok.io
# Webhook URL 设为: https://abc123.ngrok.io/webhook/wecom
```

### 步骤 4：配置 OpenClaw 企业微信应用

```json
{
  "channels": {
    "wecom-app": {
      "type": "wecom",
      "enabled": true,
      "corpid": "ww1234567890abcdef",
      "corpsecret": "your_app_secret_here",
      "agentid": "1000001",
      "token": "MyOpenClawToken123",
      "encoding_aes_key": "your_43_char_encoding_aes_key_here",
      "name": "企业微信 AI 助手",
      "options": {
        "auto_reply": true,
        "welcome_message": "👋 你好！我是 OpenClaw AI 助手，有什么可以帮你的？",
        "typing_indicator": true
      }
    }
  }
}
```

### 步骤 5：验证服务器

企业微信会发送一个 GET 请求验证服务器，OpenClaw 会自动处理：

```bash
# 启动 OpenClaw
openclaw start

# 在管理后台点击「保存」，查看 OpenClaw 日志
# 应看到类似：[WeComChannel] 服务器验证成功 ✅
```

```json

      "rules": [
        {
          "match": { "user": "admin@company.com" },
          "agent": "gpt4-assistant",
          "description": "管理员使用 GPT-4"

        {
          "match": { "department": "research" },

        {
          "match": { "all": true },
          "agent": "gpt35-assistant",
```

## 🔔 主动消息推送

### 推送给指定用户

```javascript
// 在 OpenClaw 技能或 Cron 任务中使用
const wecom = openclaw.getChannel('wecom-app');

// 推送给单个用户
await wecom.sendMessage({
  touser: 'zhangsan',  // 企业微信 userid
  msgtype: 'text',
  content: '你今天的任务提醒：请完成 Q1 报告 📋'
});

// 推送给多个用户（竖线分隔）
await wecom.sendMessage({
  touser: 'zhangsan|lisi|wangwu',
  msgtype: 'markdown',
  content: `## 📊 日报汇总\n\n今日完成任务：5 项\n待处理：3 项`
});

// 推送给整个部门
await wecom.sendMessage({
  toparty: '2',  // 部门 ID
  msgtype: 'news',
  articles: [{
    title: '本周工作总结',
    description: '点击查看本周详细工作报告',
    url: 'https://your-report-url.com',
    picurl: 'https://your-image-url.com/cover.png'
  }]
});
```

### 定时推送示例

```javascript
// cron/wecom-daily-report.js
const cron = require('node-cron');

module.exports = {
  name: 'wecom-daily-report',
  
  setup: (openclaw) => {
    // 每天早上 9 点发送日报
    cron.schedule('0 9 * * 1-5', async () => {
      const report = await generateDailyReport();
      
      await openclaw.getChannel('wecom-app').sendMessage({
        toparty: 'all',
        msgtype: 'markdown',
        content: report
      });
      
      console.log('[WeComCron] 日报发送成功');
    });
  }
};

async function generateDailyReport() {
  const date = new Date().toLocaleDateString('zh-CN');
  return `## 📅 ${date} 工作日报

### 今日重点
> 由 AI 助手自动生成

**待处理事项：**
- [ ] 检查昨日未完成任务
- [ ] 参加 10:00 晨会
- [ ] 提交周报（周五）

**今日提醒：**
如需 AI 协助，直接在企业微信中发消息即可 🤖

---
*由 OpenClaw 自动发送*`;
}
```

## 🛠️ 常见问题

### Q1: 验证服务器失败

**症状**：管理后台保存时提示「服务器验证失败」

**排查步骤**：
1. 确认服务已启动：`openclaw status`
2. 确认 URL 可访问：`curl https://your-server.com/webhook/wecom`
3. 检查 Token 是否一致
4. 查看日志：`openclaw logs --channel wecom`

```bash
# 调试模式启动
OPENCLAW_DEBUG=wecom openclaw start
```

### Q2: 收不到消息回复

**排查步骤**：
1. 检查 agentid 是否正确
2. 检查 secret 是否过期（定期轮换）
3. 确认消息接收 API 状态为「已启用」
4. 检查应用可见范围是否包含测试用户

### Q3: 发送消息提示 60020 错误

**原因**：接收用户不在应用可见范围内

**解决**：管理后台 → 应用 → 可见范围 → 添加用户/部门

### Q4: AES 解密失败

**原因**：EncodingAESKey 不匹配

**解决**：
```bash
# 重新生成配置
openclaw config --channel wecom --reset-aes-key
```

## 🎯 进阶玩法

### 审批流程机器人

```javascript
// 监听审批事件，自动处理
openclaw.on('wecom:approval', async (event) => {
  if (event.type === 'approved') {
    await notifyRequester(event.user, '你的申请已通过 ✅');
    await createTask(event.data);
  } else if (event.type === 'rejected') {
    await notifyRequester(event.user, `申请被拒绝，原因：${event.reason}`);
  }
});
```

### 智能客服路由

```javascript
// 根据问题类型分配给不同处理人
openclaw.on('wecom:message', async (message, reply) => {
  const intent = await classifyIntent(message.text);
  
  const routingMap = {
    'technical': 'tech-support-group',
    'billing': 'finance-group',
    'general': null  // 由 AI 直接回复
  };
  
  if (routingMap[intent]) {
    await forwardToHuman(message, routingMap[intent]);
  } else {
    const aiReply = await openclaw.ai.chat(message.text);
    await reply(aiReply);
  }
});
```
