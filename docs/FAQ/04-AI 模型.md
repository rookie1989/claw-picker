## 模型选择

### Q1: 应该选择哪个模型？

**推荐：**

| 场景 | 推荐模型 | 理由 |
|------|----------|------|
| 日常对话 | Claude Sonnet 4 | 平衡性能和成本 |
| 复杂推理 | Claude Opus 4 | 最强推理能力 |
| 快速响应 | Claude Haiku 3 | 速度快，成本低 |
| 代码生成 | GPT-4o | 代码能力强 |
| 中文任务 | 智谱 GLM-4 | 中文优化 |
| 预算有限 | GPT-3.5-turbo | 性价比高 |

---

### Q2: 如何切换模型？

**方法一：配置文件**
```yaml
providers:
  - name: anthropic
    apiKey: sk-ant-xxx
    defaultModel: claude-sonnet-4-20250514  # 修改这里
```

**方法二：命令行**
```bash
openclaw config set defaultModel claude-haiku-3
```

**方法三：运行时指定**
```bash
openclaw chat --model claude-opus-4 "问题"
```

---

### Q3: 如何配置多模型 Failover？

**答案：**

```yaml
providers:
  - name: anthropic
    apiKey: sk-ant-xxx
    defaultModel: claude-sonnet-4-20250514
    priority: 1  # 主模型
    
  - name: openai
    apiKey: sk-xxx
    defaultModel: gpt-4o
    priority: 2  # 备用
    
  - name: zhipu
    apiKey: xxx
    defaultModel: glm-4
    priority: 3  # 第三备用
```

当主模型失败（API 错误、超时等）时，自动切换到备用模型。

### Q5: API Key 无效/过期

**排查：**

1. 检查 Key 是否正确复制（无多余空格）
2. 确认 Key 未过期
3. 检查账户余额/配额
4. 重新生成 Key

**测试：**
```bash
openclaw provider test anthropic
```

---

### Q6: 如何安全存储 API Key？

**❌ 不要这样：**
```yaml
providers:
  - name: anthropic
    apiKey: sk-ant-xxxxxxxxxxxx  # 明文存储，危险！
```

**✅ 应该这样：**
```yaml
providers:
  - name: anthropic
    apiKey: ${ANTHROPIC_API_KEY}  # 环境变量
```

```bash
# 添加到 ~/.bashrc 或 ~/.zshrc
export ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxx
```

## 性能优化

### Q7: 响应速度慢

**优化方案：**

1. **切换到更快的模型：**
   ```yaml
   providers:
     - name: anthropic
       defaultModel: claude-haiku-3  # 最快
   ```

2. **减少上下文长度：**
   ```yaml
   context:
     maxTokens: 50000  # 降低上下文限制
   ```

3. **启用缓存：**
   ```yaml
   cache:
     enabled: true
     ttl: 3600  # 1 小时
   ```

---

### Q8: 如何降低使用成本？

**方案：**

1. **使用便宜模型：**
   ```yaml
   providers:
     - name: openai
       defaultModel: gpt-3.5-turbo  # 比 GPT-4 便宜 10 倍
   ```

2. **设置 Token 限制：**
   ```yaml
   context:
     maxTokens: 20000  # 限制最大输出
   ```

3. **启用响应压缩：**
   ```yaml
   context:
     compression: true
   ```

4. **使用缓存：**
   ```yaml
   cache:
     enabled: true
   ```

## 错误处理

### Q9: 429 Too Many Requests

**原因：** API 调用频率超限

**解决方案：**

1. **降低调用频率：**
   ```yaml
   providers:
     - name: anthropic
       rateLimit:
         requestsPerMinute: 10
   ```

2. **增加重试：**
   ```yaml
   providers:
     - name: anthropic
       retry: 5  # 重试次数
       retryDelay: 1000  # 重试间隔（毫秒）
   ```

3. **升级账户配额**

---

### Q10: 500 Internal Server Error

**原因：** 模型服务端错误

**解决方案：**

1. 等待几分钟后重试
2. 切换到备用模型
3. 检查模型状态页：
   - Anthropic: https://status.anthropic.com/
   - OpenAI: https://status.openai.com/

---

### Q11: 上下文超长

**错误：** `maximum context length exceeded`

**解决方案：**

1. **减少上下文：**
   ```yaml
   context:
     maxTokens: 50000
   ```

2. **启用自动摘要：**
   ```yaml
   context:
     summary: true
     summaryThreshold: 80000
   ```

3. **启用压缩：**
   ```yaml
   context:
     compression: true
   ```

## 高级配置

### Q14: 如何自定义模型参数？

**答案：**

```yaml
providers:
  - name: anthropic
    apiKey: sk-ant-xxx
    defaultModel: claude-sonnet-4-20250514
    params:
      temperature: 0.7      # 创造性（0-1）
      maxTokens: 4096       # 最大输出
      topP: 0.9             # 采样概率
      topK: 40              # 候选词数量
```

### Q15: 如何为不同任务使用不同模型？

**答案：**

```yaml
routing:
  enabled: true
  rules:
    - pattern: ".*(代码 | 编程|debug).*"
      model: gpt-4o
      
    - pattern: ".*(写作 | 翻译|润色).*"
      model: claude-opus-4
      
    - pattern: ".*(天气 | 搜索|查询).*"
      model: claude-haiku-3  # 快速响应
      
    - pattern: ".*"  # 默认
      model: claude-sonnet-4-20250514
```
