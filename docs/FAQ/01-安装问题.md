# FAQ - 安装问题

> 安装 OpenClaw 时遇到的常见问题解答

---

## Q1: npm install 卡住不动

**问题：** 执行 `npm install -g openclaw` 时卡在某个进度不动。

**解决方案：**

```bash
# 1. 切换 npm 镜像
npm config set registry https://registry.npmmirror.com
npm install -g openclaw

# 2. 清除缓存重试
npm cache clean --force
npm install -g openclaw

# 3. 增加超时时间
npm config set fetch-retry-mintimeout 20000
npm config set fetch-retry-maxtimeout 120000
npm install -g openclaw
```

---

## Q2: 权限错误 EPERM/EACCES

**问题：** 
```
npm ERR! Error: EPERM: operation not permitted
npm ERR! Error: EACCES: permission denied
```

**解决方案：**

**Mac/Linux:**
```bash
# 不要使用 sudo！修改 npm 目录权限
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
npm install -g openclaw
```

**Windows:**
```powershell
# 以管理员身份运行 PowerShell
npm install -g openclaw

# 或修改 npm 目录
npm config set prefix "%APPDATA%\npm"
# 将 %APPDATA%\npm 添加到 PATH
```

## Q3: git command not found

**问题：** `Error: git command not found`

**解决方案：**

**Mac:**
```bash
xcode-select --install
```

**Windows:**
下载安装 Git for Windows: https://git-scm.com/download/win

**Linux:**
```bash
# Ubuntu/Debian
sudo apt update && sudo apt install git

# CentOS/RHEL
sudo yum install git
```

---

## Q4: node-gyp 编译错误

**问题：**
```
npm ERR! gyp ERR! stack Error: not found: python
npm ERR! gyp ERR! stack Error: Can't find Python executable "python"
```

**解决方案：**

**Mac:**
```bash
xcode-select --install
```

**Windows:**
```powershell
# 安装 Python 3 和构建工具
npm install --global --production windows-build-tools
```

**Linux:**
```bash
sudo apt update
sudo apt install build-essential python3
npm config set python python3
```

## Q5: openclaw 命令找不到

**问题：** 安装完成后运行 `openclaw` 提示 `command not found`

**解决方案：**

**Mac/Linux:**
```bash
# 查找 npm 全局目录
npm bin -g

# 添加到 PATH
echo 'export PATH=$(npm bin -g):$PATH' >> ~/.bashrc
source ~/.bashrc

# 验证
which openclaw
```

**Windows:**
1. 找到 npm 安装目录（通常 `C:\Users\用户名\AppData\Roaming\npm`）
2. 添加到系统 PATH 环境变量
3. 重启终端

## Q6: 安装版本过旧

**问题：** 安装的不是最新版本

**解决方案：**

```bash
# 清除缓存
npm cache clean --force

# 强制安装最新版
npm install -g openclaw@latest

# 验证版本
openclaw --version
```

---

## Q7: SSL 证书错误

**问题：**
```
npm ERR! Error: unable to verify the first certificate
```

**解决方案：**

```bash
# 临时方案（不推荐生产环境）
npm config set strict-ssl false
npm install -g openclaw

# 或配置 CA 证书
npm config set cafile /path/to/ca-cert.pem
```

---

## Q8: Windows 中文乱码

**问题：** 命令行显示乱码

**解决方案：**

```cmd
# 临时切换编码
chcp 65001

# 或使用 Windows Terminal（推荐）
# 下载：https://aka.ms/terminal
```

---

## Q9: 磁盘空间不足

**问题：**
```
npm ERR! Error: ENOSPC: no space left on device
```

**解决方案：**

```bash
# 清理磁盘空间
df -h  # 查看空间

# 清理 npm 缓存
npm cache clean --force

# 清理旧版本
npm prune -g
```

---

## Q10: 网络超时

**问题：**
```
npm ERR! network timeout at: https://registry.npmjs.org/...
```

**解决方案：**

```bash
# 使用国内镜像
npm config set registry https://registry.npmmirror.com

# 增加超时时间
npm config set fetch-retry-mintimeout 20000
npm config set fetch-retry-maxtimeout 120000
```
