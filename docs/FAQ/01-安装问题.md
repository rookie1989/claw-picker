# ❓ 安装问题 FAQ

OpenClaw 安装过程中的常见问题解答。

## 🔧 前置要求

### Q: Node.js 版本要求？

**A:** 需要 Node.js 18 或更高版本。

检查版本：
```bash
node --version
```

升级 Node.js：
```bash
# 使用 nvm（推荐）
nvm install 18
nvm use 18

# 或直接下载
# https://nodejs.org/
```

### Q: 需要多少内存？

**A:** 
- 最低：2GB
- 推荐：4GB+
- 大型部署：8GB+

查看内存：
```bash
free -h
```

### Q: 支持哪些操作系统？

**A:** 
- ✅ Ubuntu 20.04+
- ✅ Debian 10+
- ✅ CentOS 7+
- ✅ macOS 11+
- ✅ Windows 10+ (WSL2)

## 📦 安装问题

### Q: npm install 失败？

**A:** 常见原因：

1. **网络问题**：
```bash
# 使用淘宝镜像
npm config set registry https://registry.npmmirror.com
```

2. **权限问题**：
```bash
# 使用 sudo
sudo npm install -g openclaw

# 或修复 npm 权限
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

3. **磁盘空间不足**：
```bash
df -h
```

### Q: openclaw 命令找不到？

**A:** 
```bash
# 检查 npm 全局路径
npm config get prefix

# 添加 PATH
export PATH=$(npm config get prefix)/bin:$PATH

# 永久生效
echo 'export PATH=$(npm config get prefix)/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

### Q: 安装后版本过旧？

**A:** 
```bash
# 清除缓存
npm cache clean --force

# 强制安装最新版
npm install -g openclaw@latest

# 验证版本
openclaw --version
```

## ⚙️ 配置问题

### Q: 配置文件在哪？

**A:** 
```
~/.openclaw/config.json
```

查看配置：
```bash
cat ~/.openclaw/config.json
```

### Q: 如何重置配置？

**A:** 
```bash
# 备份旧配置
mv ~/.openclaw/config.json ~/.openclaw/config.json.backup

# 重新初始化
openclaw onboard
```

### Q: 配置错误如何修复？

**A:** 
```bash
# 验证配置
openclaw config validate

# 查看错误日志
openclaw logs --level error
```

## 🚀 启动问题

### Q: 网关启动失败？

**A:** 

1. **查看详细错误**：
```bash
openclaw gateway start --verbose
```

2. **检查端口占用**：
```bash
lsof -i :3000
# 或
netstat -tulpn | grep 3000
```

3. **解决端口占用**：
```bash
# 杀死占用进程
kill -9 <PID>

# 或更改端口
openclaw config patch --path server.port --value 3001
```

### Q: 启动后立即退出？

**A:** 查看日志：
```bash
openclaw logs --tail 100
```

常见原因：
- 配置文件格式错误
- 缺少必要凭证
- 端口被占用

### Q: 如何后台运行？

**A:** 
```bash
# 使用 systemd（推荐）
systemctl enable openclaw
systemctl start openclaw

# 或使用 nohup
nohup openclaw gateway start &

# 或使用 screen
screen -S openclaw
openclaw gateway start
# Ctrl+A, D 退出屏幕
```

## 🔌 插件问题

### Q: 插件加载失败？

**A:** 
```bash
# 列出已加载插件
openclaw plugins list

# 查看插件日志
openclaw logs --grep plugin
```

### Q: 如何安装插件？

**A:** 
```bash
# 官方插件
npm install -g @openclaw/plugin-name

# 在配置中启用
{
  "plugins": {
    "entries": ["plugin-name"]
  }
}
```

### Q: 插件冲突怎么办？

**A:** 
```bash
# 禁用插件
openclaw plugins disable plugin-name

# 卸载插件
npm uninstall -g @openclaw/plugin-name
```

## 🌐 网络问题

### Q: 无法连接 API 服务？

**A:** 
```bash
# 测试网络
curl -I https://api.example.com

# 检查 DNS
nslookup api.example.com

# 更换 DNS
echo "nameserver 8.8.8.8" > /etc/resolv.conf
```

### Q: 需要配置代理？

**A:** 
```bash
# 临时代理
export HTTP_PROXY=http://proxy:port
export HTTPS_PROXY=http://proxy:port

# 永久配置
echo 'export HTTP_PROXY=http://proxy:port' >> ~/.bashrc
```

### Q: 防火墙阻止连接？

**A:** 
```bash
# Ubuntu
ufw allow 3000/tcp
ufw status

# CentOS
firewall-cmd --permanent --add-port=3000/tcp
firewall-cmd --reload
```

## 📱 移动端问题

### Q: 无法配对移动设备？

**A:** 

1. **检查网络**：确保手机和服务器在同一网络
2. **检查配置**：
```json
{
  "nodes": {
    "enabled": true
  }
}
```
3. **重启网关**：
```bash
openclaw gateway restart
```

### Q: QR 码不显示？

**A:** 
```bash
# 查看日志
openclaw logs --grep pairing

# 检查 publicUrl 配置
{
  "gateway": {
    "publicUrl": "https://your-domain.com"
  }
}
```

## 🔐 权限问题

### Q: 权限不足错误？

**A:** 
```bash
# 检查文件权限
ls -la ~/.openclaw

# 修复权限
chown -R $USER:$USER ~/.openclaw
chmod -R 700 ~/.openclaw
```

### Q: 无法访问某些功能？

**A:** 检查技能配置：
```json
{
  "skills": {
    "entries": ["required-skill"]
  }
}
```

## 💾 数据问题

### Q: 如何备份数据？

**A:** 
```bash
# 备份整个配置目录
cp -r ~/.openclaw ~/.openclaw.backup

# 或只备份重要数据
tar -czf openclaw-backup.tar.gz ~/.openclaw/config ~/.openclaw/data
```

### Q: 如何迁移到新服务器？

**A:** 
```bash
# 1. 备份
tar -czf openclaw-backup.tar.gz ~/.openclaw

# 2. 传输到新服务器
scp openclaw-backup.tar.gz user@new-server:~

# 3. 恢复
tar -xzf openclaw-backup.tar.gz -C ~

# 4. 重新安装
npm install -g openclaw

# 5. 启动
openclaw gateway start
```
