# FAQ 09：故障排查与诊断

> 当 OpenClaw 遇到问题时，系统化地定位和解决故障。

---

## Q1：OpenClaw 启动后立即退出，怎么排查？

**症状**：运行 `openclaw start` 后进程立即退出，没有任何输出。

**排查步骤**：

```bash
# 1. 查看详细日志
openclaw start --log-level debug 2>&1 | tee /tmp/openclaw.log

# 2. 检查配置文件语法
openclaw config validate

# 3. 检查端口占用
lsof -i :3000
# 如被占用，修改配置或 kill 占用进程

# 4. 检查环境变量
openclaw env check
```

**常见原因**：

| 原因 | 解决方案 |
|------|---------|
| 配置文件 YAML 语法错误 | 用在线 YAML 校验器检查 |
| 端口被占用 | 更换端口或关闭占用进程 |
| 必要环境变量缺失 | 补充 `.env` 文件 |
| 数据库连接失败 | 检查数据库配置和网络 |
| Node.js 版本过低 | 升级到 Node.js 18+ |

## Q2：AI 回复突然变慢或超时？

**症状**：平时 2-3 秒回复，突然变成 30 秒以上或直接超时。

**快速诊断**：

```bash
# 测试 AI 提供商连接
openclaw test-ai --provider openai --timeout 10

# 查看当前请求队列
openclaw status --verbose

# 检查速率限制状态
openclaw ratelimit status
```

**原因与解决**：

**原因 1：AI 提供商服务降级**
```bash
# 检查 OpenAI 状态
curl https://status.openai.com/api/v2/status.json | jq .status.description

# 临时切换备用模型
openclaw config set ai.fallback_model gpt-3.5-turbo
```

**原因 2：触发速率限制（429 错误）**
```yaml
# openclaw.config.yaml - 添加重试和退避配置
ai:
  rate_limit:
    retry_on_429: true
    max_retries: 3
    backoff_multiplier: 2    # 指数退避：1s → 2s → 4s
    initial_delay: 1000
```

**原因 3：本地网络问题**
```bash
# 检查到 OpenAI 的延迟
ping api.openai.com
curl -w "总耗时: %{time_total}s\n" -o /dev/null -s https://api.openai.com/v1/models

# 配置代理
export HTTPS_PROXY=http://your-proxy:7890
```

## Q3：消息发送成功但用户收不到？

**飞书/企业微信/钉钉消息丢失**

```bash
# 1. 检查 OpenClaw 发送日志
openclaw logs --channel feishu --level error --last 100

# 2. 检查 Webhook 有效性
curl -X POST "你的 webhook 地址" \
  -H "Content-Type: application/json" \
  -d '{"msg_type":"text","content":{"text":"测试"}}'

# 3. 验证 Token 是否过期
openclaw auth refresh --channel feishu
```

**可能原因**：
- Webhook 已失效（企业微信 Webhook 30 天无调用会过期）
- 应用权限被收回
- 用户不在应用可见范围内

## Q4：数据库连接错误 `ECONNREFUSED`？

```bash
# 检查数据库状态
systemctl status postgresql   # 或 mysql

# 测试连接
psql -h localhost -U openclaw -d openclaw_db -c "SELECT 1"

# 检查连接池配置
openclaw db status
```

```yaml
# 调整连接池参数
database:
  pool:
    min: 2
    max: 10
    acquire_timeout: 30000
    idle_timeout: 600000
    # 网络不稳定时开启重连
    reconnect: true
    reconnect_delay: 5000
```

## Q5：内存占用持续增长（内存泄漏）？

```bash
# 查看当前内存使用
openclaw status | grep memory

# 生成堆快照（需重启生效）
openclaw debug heap-snapshot --output /tmp/heap.heapsnapshot

# 使用 clinic.js 诊断
npm install -g clinic
clinic doctor -- node dist/index.js
```

**常见泄漏点**：

```javascript
// ❌ 错误：全局缓存无限增长
const cache = new Map();
// 每次都往里加，从不清理

// ✅ 正确：设置最大容量 + TTL
const LRU = require('lru-cache');
const cache = new LRU({ max: 1000, ttl: 30 * 60 * 1000 });
```

## Q6：日志报 `EMFILE: too many open files`？

```bash
# 查看当前限制
ulimit -n

# 临时提高限制
ulimit -n 65535

# 永久修改（Linux）
echo "* soft nofile 65535" >> /etc/security/limits.conf
echo "* hard nofile 65535" >> /etc/security/limits.conf
```

## Q7：如何生成诊断报告提交 Issue？

```bash
# 一键生成诊断报告（自动脱敏敏感信息）
openclaw diagnose --output /tmp/openclaw-diagnose.zip

# 报告内容包含：
# - 版本信息
# - 系统环境
# - 配置摘要（脱敏）
# - 最近 500 条错误日志
# - 性能指标快照
```

提交 Issue 时附上此报告，可大幅加快问题定位速度。
