# FAQ - 性能优化

> OpenClaw 性能调优、资源管理、速度优化相关问题解答

---

## 响应速度

### Q1: Bot 响应慢

**排查步骤：**

1. **检查模型响应时间：**
   ```bash
   openclaw chat --verbose "测试"
   # 查看各阶段耗时
   ```

2. **检查网络延迟：**
   ```bash
   ping api.anthropic.com
   ```

3. **查看 Gateway 负载：**
   ```bash
   openclaw gateway status
   ```

**优化方案：**

**切换到更快的模型：**
```yaml
providers:
  - name: anthropic
    defaultModel: claude-haiku-3  # 最快
```

**减少上下文：**
```yaml
context:
  maxTokens: 20000  # 降低上下文长度
```

**启用缓存：**
```yaml
cache:
  enabled: true
  ttl: 3600
```

---

### Q2: 首次响应特别慢

**原因：** 模型冷启动或上下文加载

**解决方案：**

1. **预热模型：**
   ```bash
   # 定期发送简单请求保持活跃
   */5 * * * * openclaw chat "ping"
   ```

2. **减少初始上下文：**
   ```yaml
   context:
     initialMessages: 10  # 只加载最近 10 条
   ```

3. **使用流式响应：**
   ```yaml
   gateway:
     streaming: true  # 边生成边返回
   ```

## 内存管理

### Q3: 内存占用过高

**排查：**
```bash
# 查看内存使用
ps aux | grep openclaw

# 或使用 Dashboard
openclaw dashboard
```

**优化方案：**

**限制最大内存：**
```yaml
gateway:
  maxMemory: 2GB  # 根据服务器配置调整
```

**启用垃圾回收：**
```yaml
gateway:
  gcInterval: 300000  # 5 分钟 GC
```

**减少并发：**
```yaml
gateway:
  maxConcurrentRequests: 10
```

**清理记忆：**
```yaml
memory:
  maxMessages: 50   # 减少保留消息数
  maxDays: 3        # 减少保留天数
```

---

### Q4: 内存泄漏

**症状：** 内存使用持续增长，不下降

**解决方案：**

1. **更新到最新版本：**
   ```bash
   npm install -g openclaw@latest
   ```

2. **定期重启：**
   ```bash
   # 每天凌晨 3 点重启
   0 3 * * * openclaw gateway restart
   ```

3. **启用自动重启：**
   ```yaml
   gateway:
     autoRestart:
       enabled: true
       interval: 86400000  # 24 小时
       memoryThreshold: 1.5GB  # 内存超过阈值时重启
   ```

## CPU 优化

### Q5: CPU 占用过高

**排查：**
```bash
# 查看 CPU 使用
top -pid $(pgrep -f openclaw)
```

**优化方案：**

**限制 CPU 使用：**
```yaml
gateway:
  maxCpu: 80  # 最大 CPU 使用率%
```

**减少并发处理：**
```yaml
gateway:
  maxConcurrentRequests: 5
  requestQueue:
    enabled: true
    maxSize: 100
```

**禁用不必要的技能：**
```yaml
skills:
  - name: browser
    enabled: false  # 浏览器占用高
  - name: search
    enabled: false
```

## 网络优化

### Q6: API 调用慢

**原因：** 网络延迟或限流

**优化方案：**

**使用国内镜像：**
```yaml
providers:
  - name: zhipu  # 智谱 AI，国内访问快
    apiKey: xxx
    priority: 1
    
  - name: anthropic
    apiKey: xxx
    priority: 2  # 备用
```

**配置代理：**
```yaml
gateway:
  proxy:
    enabled: true
    http: http://your-proxy:port
    https: http://your-proxy:port
```

**启用连接池：**
```yaml
providers:
  - name: anthropic
    connectionPool:
      enabled: true
      size: 10
      idleTimeout: 30000
```

### Q7: 渠道消息延迟

**排查：**

1. 检查渠道连接状态：
   ```bash
   openclaw channel list
   ```

2. 查看渠道日志：
   ```bash
   cat ~/.openclaw/logs/channels.log
   ```

**优化：**

```yaml
channels:
  - type: telegram
    polling:
      interval: 500  # 降低轮询间隔（毫秒）
      timeout: 30
    
  - type: feishu
    webhook:
      enabled: true  # 使用 Webhook 代替轮询
```

## 存储优化

### Q8: 磁盘空间不足

**清理方案：**

```bash
# 清理日志
find ~/.openclaw/logs -name "*.log" -mtime +7 -delete

# 清理缓存
rm -rf ~/.openclaw/cache/*

# 清理旧记忆
find ~/.openclaw/memory -mtime +30 -delete
```

**配置日志轮转：**
```yaml
logging:
  file:
    maxSize: 10MB
    maxFiles: 5
```

**配置存储限制：**
```yaml
memory:
  maxStorage: 500MB
  
cache:
  maxSize: 100MB
```

---

### Q9: 数据库性能下降

**原因：** 数据量过大

**优化：**

**定期清理：**
```yaml
maintenance:
  enabled: true
  schedule: '0 3 * * *'  # 每天凌晨 3 点
  tasks:
    - cleanupMemory
    - cleanupCache
    - vacuumDatabase
```

**使用 Redis：**
```yaml
memory:
  provider: redis
  config:
    host: localhost
    port: 6379
```

## 并发处理

### Q10: 多用户场景性能下降

**优化方案：**

**增加 worker 数量：**
```yaml
gateway:
  workers: 4  # 根据 CPU 核心数调整
```

**配置请求队列：**
```yaml
gateway:
  requestQueue:
    enabled: true
    maxSize: 1000
    timeout: 30000
```

**限流保护：**
```yaml
security:
  rateLimit:
    enabled: true
    maxRequests: 100
    windowMs: 60000
    perUser: true  # 按用户限流
```

---

### Q11: 批量处理优化

**场景：** 需要处理大量消息或任务

**方案：**

**批处理配置：**
```yaml
gateway:
  batch:
    enabled: true
    size: 10       # 每批 10 个
    interval: 1000  # 1 秒间隔
```

**优先级队列：**
```yaml
gateway:
  priorityQueue:
    enabled: true
    levels:
      - name: high
        pattern: ".*(紧急 | 重要|priority).*"
      - name: normal
        pattern: ".*"
```

## 监控与诊断

### Q12: 如何监控性能？

**内置监控：**

```bash
# 查看 Gateway 状态
openclaw gateway status

# 查看性能指标
openclaw metrics

# 查看实时日志
openclaw logs --follow
```

**配置监控：**
```yaml
monitoring:
  enabled: true
  metrics:
    - responseTime
    - memoryUsage
    - cpuUsage
    - requestCount
  alert:
    - metric: memoryUsage
      threshold: 80%
      action: restart
```

---

### Q13: 性能分析工具

**内置分析：**
```bash
# 生成性能报告
openclaw diagnose --report

# 分析慢请求
openclaw logs --slow --threshold 5000  # >5 秒的请求
```

**外部工具：**

```bash
# 使用 Node.js 性能工具
node --inspect openclaw

# 然后在 Chrome 访问 chrome://inspect
```

## 部署优化

### Q14: 生产环境配置建议

**推荐配置：**

```yaml
# 生产环境配置
gateway:
  port: 8080
  host: 0.0.0.0
  workers: 4
  maxMemory: 4GB
  maxCpu: 80
  
  # 启用集群模式
  cluster:
    enabled: true
    instances: 4
  
  # 健康检查
  healthCheck:
    enabled: true
    interval: 30000

logging:
  level: warn  # 生产环境减少日志
  format: json
  output:
    - file

security:
  rateLimit:
    enabled: true
    maxRequests: 100
    windowMs: 60000
```

### Q15: Docker 部署优化

**Docker Compose 配置：**

```yaml
version: '3'
services:
  openclaw:
    image: openclaw/openclaw:latest
    deploy:
      resources:
        limits:
          cpus: '2'
          memory: 2G
        reservations:
          cpus: '1'
          memory: 1G
    environment:
      - NODE_ENV=production
      - ANTHROPIC_API_KEY=${ANTHROPIC_API_KEY}
    volumes:
      - ./config:/app/config
      - ./data:/app/data
    restart: unless-stopped
```
