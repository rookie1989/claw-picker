## 💾 内存管理

### Q: 内存占用过高？

**A:** 

1. **限制会话数量**：
```json
{
  "session": {
    "maxSessions": 10
  }
}
```

2. **定期清理旧会话**：
```bash
openclaw sessions cleanup
```

3. **添加 Swap**（服务器）：
```bash
fallocate -l 2G /swapfile
mkswap /swapfile
swapon /swapfile
```

4. **重启网关**：
```bash
openclaw gateway restart
```

### Q: 如何监控内存使用？

**A:** 
```bash
# 实时查看
htop

# 查看 OpenClaw 进程
ps aux | grep openclaw

# 查看内存详情
free -h
```

## 📊 数据库优化

### Q: 数据查询慢？

**A:** 

1. **添加索引**（如果使用数据库）
2. **限制查询范围**：
```javascript
// 优化前
const all = await db.findAll();

// 优化后
const recent = await db.find({limit: 100, order: 'desc'});
```

3. **使用缓存**：
```javascript
const cache = new Map();
async function getData(key) {
  if (cache.has(key)) return cache.get(key);
  const data = await fetch(key);
  cache.set(key, data);
  return data;
}
```

{
  "monitoring": {

    "metrics": ["cpu", "memory", "response_time"],

### Q: 如何查看性能日志？

**A:** 
```bash
# 查看响应时间
openclaw logs --grep "response"

# 查看错误
openclaw logs --level error

# 实时监控
openclaw logs --follow | grep -E "error|slow"
```

## 🐛 性能问题排查

### Q: 如何定位性能瓶颈？

**A:** 

1. **查看日志**：
```bash
openclaw logs --tail 1000 | grep -E "slow|timeout"
```

2. **监控资源**：
```bash
htop
iotop
nethogs
```

3. **分析请求**：
```javascript
console.time('request');
await handleRequest();
console.timeEnd('request');
```

### Q: 系统变慢如何恢复？

**A:** 
```bash
# 1. 重启网关
openclaw gateway restart

# 2. 清理缓存
openclaw sessions cleanup

# 3. 清理日志
openclaw logs --clear

# 4. 检查资源
free -h
df -h
```
