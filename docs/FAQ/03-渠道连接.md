# FAQ - 渠道连接

> Telegram、WhatsApp、飞书、Discord 等渠道连接问题解答

---

## Telegram

### Q1: 如何创建 Telegram Bot？

**步骤：**

1. 在 Telegram 搜索 `@BotFather`
2. 发送 `/newbot`
3. 按提示输入 Bot 名称和用户名
4. 获取 Token（格式：`123456789:ABCdefGHIjklMNOpqrsTUVwxyz`）

**配置：**
```yaml
channels:
  - type: telegram
    token: 123456789:ABCdefGHIjklMNOpqrsTUVwxyz
```

---

### Q2: Bot 收不到消息

**排查步骤：**

1. 检查 Token 是否正确
2. 确认已启动 Bot（在 Telegram 中点击"Start"）
3. 检查白名单配置：
   ```yaml
   channels:
     - type: telegram
       token: YOUR_TOKEN
       allowFrom:
         - username: your_username  # 确保用户名正确
   ```
4. 查看 Gateway 日志：`cat ~/.openclaw/logs/gateway.log`

---

### Q3: Bot 无法回复消息

**原因：** 权限不足或配置错误

**解决方案：**

1. 检查 Bot 是否是群组成员
2. 检查 Bot 是否有发送消息权限
3. 确认配置中未禁用回复：
   ```yaml
   channels:
     - type: telegram
       token: YOUR_TOKEN
       replyEnabled: true
   ```

---

### Q4: 群组消息被忽略

**原因：** 未配置 mentionOnly 或白名单

**解决方案：**

```yaml
channels:
  - type: telegram
    token: YOUR_TOKEN
    mentionOnly: false  # 响应所有消息
    # 或
    mentionOnly: true   # 只响应 @Bot 的消息
```

## WhatsApp

### Q1: 如何配对 WhatsApp？

**步骤：**

1. 运行命令：`openclaw channel add whatsapp`
2. 扫描显示的二维码
3. 用手机 WhatsApp 扫描二维码
4. 等待配对完成

**注意：** 配对信息保存在 `~/.openclaw/whatsapp/` 目录

---

### Q2: 配对失败

**常见原因：**

1. 网络问题（无法连接 WhatsApp 服务器）
2. 二维码过期（重新运行命令）
3. 端口被占用

**解决方案：**

```bash
# 清除旧配对
rm -rf ~/.openclaw/whatsapp

# 重新配对
openclaw channel add whatsapp
```

---

### Q3: 收不到消息

**排查：**

1. 检查 WhatsApp 是否在线
2. 确认 Bot 已添加到联系人
3. 查看会话状态：
   ```bash
   openclaw channel list
   ```

---

## 飞书

### Q1: 如何创建飞书应用？

**步骤：**

1. 访问 https://open.feishu.cn/app
2. 点击"创建应用"
3. 获取 App ID 和 App Secret
4. 配置事件订阅和机器人权限

**配置：**
```yaml
channels:
  - type: feishu
    appId: cli_xxxxxxxxx
    appSecret: xxxxxxxxxxxxxx
```

---

### Q2: 事件订阅失败

**原因：** 回调地址配置错误

**解决方案：**

1. 确保 Gateway 可公网访问
2. 在飞书应用后台配置回调 URL：
   ```
   https://your-domain.com/api/feishu/callback
   ```
3. 使用内网穿透工具（如 ngrok）测试：
   ```bash
   ngrok http 8080
   ```

---

### Q3: 机器人无法@成员

**原因：** 权限不足

**解决方案：**

1. 在飞书应用后台添加权限：
   - 获取用户信息
   - 发送消息
   - @成员
2. 重新发布应用

---

## Discord

### Q1: 如何创建 Discord Bot？

**步骤：**

1. 访问 https://discord.com/developers/applications
2. 创建新应用
3. 在"Bot"页面创建 Bot
4. 获取 Token
5. 在"OAuth2"页面生成邀请链接

**配置：**
```yaml
channels:
  - type: discord
    botToken: YOUR_BOT_TOKEN
    guildId: "123456789"  # 服务器 ID
```

---

### Q2: Bot 无法加入服务器

**解决方案：**

1. 生成正确的邀请链接：
   ```
   https://discord.com/api/oauth2/authorize?client_id=YOUR_CLIENT_ID&permissions=8&scope=bot
   ```
2. 确保有"管理员"权限
3. 检查服务器是否启用了机器人加入

---

### Q3: 收不到群组消息

**原因：** 缺少 Message Content Intent

**解决方案：**

1. 在 Discord 开发者后台
2. 进入应用 → Bot → Privileged Gateway Intents
3. 启用 "Message Content Intent"
4. 保存并重启 Bot

## 通用问题

### Q1: 如何测试渠道连接？

**命令：**
```bash
# 列出所有渠道
openclaw channel list

# 测试特定渠道
openclaw channel test telegram

# 发送测试消息
openclaw channel send telegram --to @username "测试消息"
```

---

### Q2: 如何切换渠道？

**答案：**

```bash
# 添加新渠道
openclaw channel add telegram --token NEW_TOKEN

# 移除旧渠道
openclaw channel remove telegram

# 修改配置后重启
openclaw gateway restart
```

---

### Q3: 如何配置多个渠道？

**答案：**

```yaml
channels:
  - type: telegram
    token: TELEGRAM_TOKEN
    
  - type: feishu
    appId: cli_xxx
    appSecret: xxx
    
  - type: discord
    botToken: DISCORD_TOKEN
```

所有渠道会同时工作，消息互通。

---

### Q4: 渠道频繁断开连接

**原因：**

1. 网络不稳定
2. Token/凭证过期
3. 服务器资源不足

**解决方案：**

1. 检查网络连接
2. 重新生成 Token
3. 增加 Gateway 资源：
   ```yaml
   gateway:
     maxConnections: 100
     heartbeatInterval: 30
   ```

---

### Q5: 消息延迟高

**排查：**

1. 检查网络延迟
2. 查看 Gateway 负载：`openclaw gateway status`
3. 检查日志中的错误信息
4. 考虑升级服务器配置
