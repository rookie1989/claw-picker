# FAQ - 配置问题

> OpenClaw 配置相关的常见问题解答

---

## Q1: 配置文件在哪里？

**答案：**

| 系统 | 路径 |
|------|------|
| Mac/Linux | `~/.openclaw/config.yaml` |
| Windows | `C:\Users\用户名\.openclaw\config.yaml` |

查看配置：
```bash
cat ~/.openclaw/config.yaml
```

---

## Q2: 如何切换模型？

**答案：**

**方法一：修改配置文件**
```yaml
providers:
  - name: anthropic
    apiKey: sk-ant-xxx
    defaultModel: claude-sonnet-4-20250514  # 改这里
```

**方法二：命令行**
```bash
openclaw config set defaultModel claude-haiku-3
```

**方法三：运行时指定**
```bash
openclaw chat --model claude-opus-4 "问题"
```

---

## Q3: 如何配置多个模型？

**答案：**

```yaml
providers:
  - name: anthropic
    apiKey: sk-ant-xxx
    defaultModel: claude-sonnet-4-20250514
    priority: 1  # 主模型
    
  - name: openai
    apiKey: sk-xxx
    defaultModel: gpt-4o
    priority: 2  # 备用
    
  - name: zhipu
    apiKey: xxx
    defaultModel: glm-4
    priority: 3  # 第三备用
```

当主模型失败时，自动切换到备用模型。

## Q4: API Key 如何安全存储？

**答案：**

**❌ 不要这样（明文）：**
```yaml
providers:
  - name: anthropic
    apiKey: sk-ant-xxxxxxxxxxxx  # 危险！
```

**✅ 应该这样（环境变量）：**
```yaml
providers:
  - name: anthropic
    apiKey: ${ANTHROPIC_API_KEY}
```

```bash
# 添加到 ~/.bashrc 或 ~/.zshrc
export ANTHROPIC_API_KEY=sk-ant-xxxxxxxxxxxx
```

## Q5: 配置文件格式错误

**问题：** 启动时提示 YAML 解析错误

**解决方案：**

1. 使用在线工具验证 YAML：https://www.yamllint.com/
2. 检查缩进（使用空格，不要用 Tab）
3. 检查特殊字符是否需要引号
4. 使用最小配置测试：

```yaml
# 最小可用配置
providers:
  - name: anthropic
    apiKey: sk-ant-xxx
```

## Q6: 如何禁用某些技能？

**答案：**

```yaml
skills:
  - name: browser
    enabled: false  # 禁用浏览器
    
  - name: search
    enabled: true   # 启用搜索
    
  - name: files
    enabled: false  # 禁用文件操作
```

---

## Q7: 如何配置技能触发器？

**答案：**

```yaml
skills:
  - name: weather
    script: ./skills/weather.js
    triggers:
      # 关键词触发
      - keywords: ["天气", "气温", "多少度"]
      
      # 正则表达式触发
      - regex: "(\\d+) 天后天气"
      
      # 命令触发
      - command: "/weather"
```

## Q8: 如何限制 Bot 只响应特定用户？

**答案：**

**Telegram:**
```yaml
channels:
  - type: telegram
    token: YOUR_TOKEN
    allowFrom:
      - username: your_username
      - user_id: 123456789
```

**飞书:**
```yaml
channels:
  - type: feishu
    appId: cli_xxx
    appSecret: xxx
    allowFrom:
      - open_id: ou_xxxxxxxxx
```

## Q9: 如何配置只响应 @ 消息？

**答案：**

```yaml
channels:
  - type: telegram
    token: YOUR_TOKEN
    mentionOnly: true  # 只响应 @Bot 的消息
```

## Q10: 如何修改 Gateway 端口？

**答案：**

```yaml
gateway:
  port: 8081  # 默认 8080
  host: 0.0.0.0  # 监听所有接口
```

或命令行：
```bash
openclaw gateway start --port 8081
```

## Q11: 记忆功能如何配置？

**答案：**

**本地记忆：**
```yaml
memory:
  enabled: true
  provider: local
  maxMessages: 100  # 保留 100 条消息
  maxDays: 7        # 保留 7 天
  config:
    dataPath: ~/.openclaw/memory
```

**Redis 记忆（多实例共享）：**
```yaml
memory:
  enabled: true
  provider: redis
  config:
    host: localhost
    port: 6379
    password: your_password
```

## Q12: 如何配置日志？

**答案：**

**开发环境（详细日志）：**
```yaml
logging:
  level: debug
  format: text
  output:
    - console
    - file
```

**生产环境（精简日志）：**
```yaml
logging:
  level: warn
  format: json
  output:
    - file
  file:
    path: /var/log/openclaw/gateway.log
    maxSize: 50MB
    maxFiles: 10
```

## Q13: 配置文件修改后需要重启吗？

**答案：**

是的，需要重启 Gateway：

```bash
# 重启
openclaw gateway restart

# 或先停止再启动
openclaw gateway stop
openclaw gateway start
```

## Q14: 如何备份配置？

**答案：**

```bash
# 备份
cp ~/.openclaw/config.yaml ~/.openclaw/config.yaml.backup.$(date +%Y%m%d)

# 恢复
cp ~/.openclaw/config.yaml.backup.20260318 ~/.openclaw/config.yaml
```

---

## Q15: 配置不生效怎么办？

**排查步骤：**

1. 检查配置文件路径是否正确
2. 验证 YAML 格式：`cat ~/.openclaw/config.yaml | yq`
3. 查看 Gateway 日志：`cat ~/.openclaw/logs/gateway.log`
4. 重启 Gateway：`openclaw gateway restart`
5. 使用最小配置测试
