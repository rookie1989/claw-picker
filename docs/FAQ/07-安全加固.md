## 认证授权

### Q1: 如何启用 API 认证？

**配置：**

```yaml
gateway:
  auth:
    enabled: true
    type: token  # 或 jwt, oauth2
    token: ${API_TOKEN}  # 从环境变量读取
```

**使用：**
```bash
# 带认证调用
curl -H "Authorization: Bearer YOUR_TOKEN" http://localhost:8080/api/chat
```

---

### Q2: 如何配置多用户权限？

**方案：**

```yaml
auth:
  enabled: true
  users:
    - username: admin
      password: ${ADMIN_PASSWORD}
      roles:
        - admin
        - user
      permissions:
        - "*"  # 所有权限
        
    - username: user1
      password: ${USER1_PASSWORD}
      roles:
        - user
      permissions:
        - chat
        - skills.basic
        - files.read
        
    - username: guest
      password: ${GUEST_PASSWORD}
      roles:
        - guest
      permissions:
        - chat
```

---

### Q3: 如何限制用户只能访问特定渠道？

**配置：**

```yaml
channels:
  - type: telegram
    token: YOUR_TOKEN
    allowFrom:
      - username: admin
      - username: user1
      
  - type: feishu
    appId: cli_xxx
    appSecret: xxx
    allowFrom:
      - open_id: ou_admin
    requireRole: admin  # 只允许管理员
```

## 命令安全

### Q4: 如何禁止危险命令？

**配置：**

```yaml
security:
  # 白名单（只允许这些命令）
  allowCommands:
    - ls
    - cat
    - grep
    - find
    - head
    - tail
    
  # 黑名单（禁止这些命令）
  denyCommands:
    - rm
    - sudo
    - chmod
    - chown
    - curl
    - wget
    - nc
    - netcat
    
  # 正则表达式匹配
  denyPatterns:
    - ".*rm.*-rf.*"
    - ".*sudo.*"
    - ".*chmod.*777.*"
```

---

### Q5: 如何限制文件访问范围？

**配置：**

```yaml
skills:
  - name: files
    enabled: true
    config:
      # 允许的目录
      allowedDirs:
        - ~/Documents
        - /shared/project
        - ~/.openclaw/data
        
      # 禁止的目录
      denyDirs:
        - /etc
        - /var
        - /root
        - /System
        
      # 文件类型限制
      allowedExtensions:
        - .txt
        - .md
        - .json
        - .yaml
        - .js
        
      # 大小限制
      maxFileSize: 10MB
      maxReadSize: 1MB  # 单次读取最大
```

## 数据安全

### Q6: 如何保护敏感数据？

**方案：**

**1. 使用环境变量：**
```yaml
providers:
  - name: anthropic
    apiKey: ${ANTHROPIC_API_KEY}  # ✅
    # apiKey: sk-ant-xxx  # ❌ 不要明文
```

**2. 加密配置文件：**
```bash
# 使用 age 加密
age -o config.yaml.age -R recipients.txt config.yaml

# 解密使用
age -d -i identity.txt config.yaml.age > config.yaml
```

**3. 使用密钥管理服务：**
```yaml
providers:
  - name: anthropic
    apiKey: ${vault:secret/openclaw/anthropic-key}
```

### Q7: 如何防止数据泄露？

**配置：**

```yaml
security:
  # 禁用日志中的敏感信息
  redactLogs: true
  redactPatterns:
    - "sk-ant-.*"  # API Key
    - "password.*"
    - "token.*"
    
  # 限制输出长度
  maxOutputLength: 10000
  
  # 禁止外部访问
  allowInternalOnly: true
  
  # 审计日志
  auditLog:
    enabled: true
    path: ~/.openclaw/logs/audit.log
    events:
      - login
      - command
      - fileAccess
```

## 网络安全

### Q8: 如何配置防火墙？

**推荐规则：**

```bash
# 只允许本地访问
ufw allow from 127.0.0.1 to any port 8080

# 或只允许特定 IP
ufw allow from 192.168.1.0/24 to any port 8080

# 禁止外部访问
ufw deny 8080
```

**配置 Gateway 监听：**
```yaml
gateway:
  host: 127.0.0.1  # 只监听本地
  # host: 0.0.0.0  # ❌ 监听所有接口（危险）
```

---

### Q9: 如何启用 HTTPS？

**方案一：使用 Nginx 反向代理**

```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;
    
    ssl_certificate /path/to/cert.pem;
    ssl_certificate_key /path/to/key.pem;
    
    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

**方案二：内置 HTTPS**

```yaml
gateway:
  https:
    enabled: true
    cert: /path/to/cert.pem
    key: /path/to/key.pem
```

### Q10: 如何防止 DDoS 攻击？

**配置：**

```yaml
security:
  # 限流
  rateLimit:
    enabled: true
    maxRequests: 100
    windowMs: 60000  # 每分钟 100 次
    perUser: true    # 按用户限流
    perIp: true      # 按 IP 限流
    
  # 连接限制
  connectionLimit:
    enabled: true
    maxConnections: 100
    perIp: 10
    
  # 黑名单
  blacklist:
    enabled: true
    ips:
      - 1.2.3.4
      - 5.6.7.8
    autoBan:
      enabled: true
      threshold: 100  # 超过 100 次请求自动封禁
      duration: 3600  # 封禁 1 小时
```

## 审计日志

### Q11: 如何启用审计日志？

**配置：**

```yaml
logging:
  audit:
    enabled: true
    path: ~/.openclaw/logs/audit.log
    format: json
    events:
      - authentication
      - authorization
      - commandExecution
      - fileAccess
      - skillExecution
      - configChange
```

**查看审计日志：**
```bash
# 查看所有事件
cat ~/.openclaw/logs/audit.log | jq

# 查看特定用户
cat ~/.openclaw/logs/audit.log | jq 'select(.user == "admin")'

# 查看命令执行
cat ~/.openclaw/logs/audit.log | jq 'select(.event == "commandExecution")'
```

### Q12: 如何监控异常行为？

**配置：**

```yaml
security:
  monitoring:
    enabled: true
    
    # 异常检测
    anomalyDetection:
      enabled: true
      rules:
        - name: highRequestRate
          condition: "requestsPerMinute > 1000"
          action: alert
          
        - name: suspiciousCommand
          pattern: ".*(rm.*-rf|sudo|chmod).*"
          action: block_and_alert
          
        - name: unusualTime
          timeRange: "00:00-06:00"
          action: alert
    
    # 告警配置
    alert:
      enabled: true
      channels:
        - type: email
          to: admin@example.com
        - type: telegram
          chatId: "123456789"
```

## 备份恢复

### Q13: 如何备份配置和数据？

**备份脚本：**

```bash
#!/bin/bash
# backup.sh

BACKUP_DIR=~/backups/openclaw
DATE=$(date +%Y%m%d_%H%M%S)

mkdir -p $BACKUP_DIR

# 备份配置
cp -r ~/.openclaw/config.yaml $BACKUP_DIR/config_$DATE.yaml

# 备份数据
tar -czf $BACKUP_DIR/data_$DATE.tar.gz ~/.openclaw/data

# 备份记忆
tar -czf $BACKUP_DIR/memory_$DATE.tar.gz ~/.openclaw/memory

# 删除 30 天前的备份
find $BACKUP_DIR -mtime +30 -delete

echo "备份完成：$BACKUP_DIR"
```

**定时备份：**
```bash
# 每天凌晨 2 点备份
0 2 * * * /path/to/backup.sh
```


```bash
tar -xzf ~/backups/openclaw/data_20260318.tar.gz -C ~/
```

```yaml
gateway:
```
