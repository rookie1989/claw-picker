# ❓ 安全加固 FAQ

OpenClaw 安全配置、权限管理、数据保护相关问题解答。

## 🔐 访问控制

### Q: 如何限制访问用户？

**A:** 
```json
{
  "security": {
    "allowlist": ["+86138****", "@trusted_user", "ou_xxx"]
  }
}
```

**支持格式**：
- 手机号：`+86138****`
- 用户名：`@username`
- 用户 ID：`ou_xxx`

### Q: 如何禁止特定用户？

**A:** 
```json
{
  "security": {
    "blocklist": ["+1234567890", "@spam_user"]
  }
}
```

### Q: 如何配置速率限制？

**A:** 
```json
{
  "security": {
    "rateLimit": {
      "requests": 100,
      "window": 3600
    }
  }
}
```

**参数说明**：
- `requests`: 允许请求次数
- `window`: 时间窗口（秒）

## 🔑 密钥管理

### Q: 如何安全存储 API 密钥？

**A:** 

**方法 1：环境变量**（推荐）
```bash
export OPENAI_API_KEY=sk-xxx
export GITHUB_TOKEN=ghp_xxx
```

**方法 2：使用 1Password**
```json
{
  "plugins": {
    "1password": {
      "vault": "Personal",
      "item": "API Keys"
    }
  }
}
```

**方法 3：加密配置文件**
```bash
# 设置文件权限
chmod 600 ~/.openclaw/config.json

# 加密存储
gpg -c ~/.openclaw/config.json
```

### Q: API 密钥泄露怎么办？

**A:** 
1. **立即撤销**：
   - OpenAI: https://platform.openai.com/api-keys
   - GitHub: https://github.com/settings/tokens
   - 其他平台类似

2. **重新生成**新密钥

3. **更新配置**

4. **检查日志**是否有异常使用

## 🛡️ 网络安全

### Q: 如何启用 HTTPS？

**A:** 

**方法 1：使用 Nginx 反向代理**（推荐）
```nginx
server {
    listen 443 ssl;
    server_name your-domain.com;
    
    ssl_certificate /etc/letsencrypt/live/your-domain.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/your-domain.com/privkey.pem;
    
    location / {
        proxy_pass http://localhost:3000;
    }
}
```

**方法 2：内置 HTTPS**
```json
{
  "server": {
    "https": true,
    "cert": "/path/to/cert.pem",
    "key": "/path/to/key.pem"
  }
}
```

### Q: 如何配置防火墙？

**A:** 
```bash
# Ubuntu (UFW)
ufw allow 22/tcp    # SSH
ufw allow 443/tcp   # HTTPS
ufw deny 3000/tcp   # 内部访问
ufw enable

# CentOS (Firewalld)
firewall-cmd --permanent --add-service=ssh
firewall-cmd --permanent --add-service=https
firewall-cmd --permanent --remove-port=3000/tcp
firewall-cmd --reload
```

### Q: 如何防止 DDoS 攻击？

**A:** 
```json
{
  "security": {
    "rateLimit": {
      "requests": 50,
      "window": 60,
      "perIp": true
    },
    "blocklist": ["known_bad_ips"]
  }
}
```

**额外措施**：
- 使用 Cloudflare 等 CDN
- 配置 Fail2Ban
- 限制连接数

### Q: 如何加密存储数据？

**A:** 
```javascript
const crypto = require('crypto');

function encrypt(text) {
  const cipher = crypto.createCipher('aes-256-cbc', process.env.ENCRYPTION_KEY);
  return cipher.update(text, 'utf8', 'hex') + cipher.final('hex');
}

function decrypt(hash) {
  const decipher = crypto.createDecipher('aes-256-cbc', process.env.ENCRYPTION_KEY);
  return decipher.update(hash, 'hex', 'utf8') + decipher.final('utf8');
}
```

### Q: 如何安全备份？

**A:** 
```bash
# 加密备份
tar -czf - ~/.openclaw | openssl enc -aes-256-cbc -salt -out backup.tar.gz.enc

# 安全传输
scp backup.tar.gz.enc user@remote:/backup/

# 定期清理旧备份
find /backup -name "*.enc" -mtime +30 -delete
```

## 🛠️ 安全工具

### Q: 如何安装 Fail2Ban？

**A:** 
```bash
# 安装
apt install fail2ban -y

# 配置
cat > /etc/fail2ban/jail.local <<EOF
[sshd]
enabled = true
bantime = 3600
maxretry = 5

[openclaw]
enabled = true
filter = openclaw
logpath = /root/.openclaw/logs/*.log
bantime = 3600
maxretry = 10
EOF

# 启动
systemctl restart fail2ban
```

### Q: 如何配置安全扫描？

**A:** 
```bash
# 定期安全检查
cat > /usr/local/bin/openclaw-security-check <<EOF
#!/bin/bash
echo "=== 安全检查 ==="
echo "1. 检查文件权限"
ls -la ~/.openclaw/config.json

echo "2. 检查开放端口"
netstat -tulpn | grep openclaw

echo "3. 检查日志错误"
grep -i error ~/.openclaw/logs/*.log | tail -20

echo "4. 检查资源使用"
free -h
df -h
EOF

chmod +x /usr/local/bin/openclaw-security-check
```

## 🐛 安全问题处理

### Q: 发现安全漏洞怎么办？

**A:** 
1. **立即隔离**受影响系统
2. **收集证据**（日志、时间线）
3. **修复漏洞**
4. **通知相关人员**
5. **事后分析**

### Q: 如何报告安全问题？

**A:** 
- GitHub Issues（非敏感）
- 邮件报告（敏感）
- 不要公开披露未修复漏洞
