# ❓ OpenClaw 常见问题 - 新增

## 🚀 性能优化

### Q: OpenClaw 启动很慢怎么办？

**A:** 检查以下几点：
1. 网络问题：确保能访问 API 服务
2. 配置过多：精简 `plugins.entries` 只保留必需的
3. 内存不足：检查系统内存，建议 2GB+

优化建议：
```json
{
  "plugins": {
    "entries": ["telegram", "weather"] // 只启用必需的
  }
}
```

### Q: 消息响应延迟高？

**A:** 可能原因：
1. 模型 API 响应慢 → 切换到更快的模型
2. 网络延迟 → 检查网络连接
3. 并发过高 → 限制并发请求数

```json
{
  "model": {
    "default": "qwen3.5-lite", // 更快的模型
    "timeout": 30000
  }
}
```

### Q: 如何减少 Token 消耗？

**A:** 
1. 精简系统提示词
2. 限制历史消息数量
3. 使用摘要代替完整历史

```json
{
  "session": {
    "maxMessages": 50, // 限制历史消息
    "summarize": true  // 启用摘要
  }
}
```

## 🔧 技能开发

### Q: 自定义技能不生效？

**A:** 检查清单：
- [ ] 技能目录在 `~/.openclaw/skills/` 下
- [ ] 包含 `SKILL.md` 文件
- [ ] `SKILL.md` 格式正确
- [ ] 在配置中启用技能
- [ ] 重启网关

调试步骤：
```bash
# 查看技能加载日志
openclaw logs --grep skill

# 验证技能文件
ls -la ~/.openclaw/skills/your-skill/
```

### Q: 技能如何调用外部 API？

**A:** 使用 `fetch` 或 `axios`：

```javascript
// skill.js
const response = await fetch('https://api.example.com/data', {
  headers: {
    'Authorization': `Bearer ${process.env.API_KEY}`
  }
});
const data = await response.json();
return data.result;
```

### Q: 技能如何访问文件系统？

**A:** 使用 Node.js `fs` 模块：

```javascript
const fs = require('fs').promises;

const content = await fs.readFile('/path/to/file.txt', 'utf8');
await fs.writeFile('/path/to/output.txt', content);
```

## 📱 移动端

### Q: 如何配置 iOS/Android 节点？

**A:** 
1. 下载 OpenClaw 移动端 App
2. 在网关运行 `openclaw node pairing`
3. 扫描 QR 码或输入配对码
4. 完成配对

常见问题：
- QR 码不显示 → 检查 `gateway.bind` 配置
- 配对失败 → 确保在同一网络
- 连接断开 → 检查防火墙设置

### Q: 移动端能做什么？

**A:** 
- ✅ 查看摄像头
- ✅ 屏幕录制
- ✅ 位置获取
- ✅ 通知管理
- ✅ 运行命令

示例：
```bash
# 查看手机摄像头
/nodes camera_snap

# 获取位置
/nodes location_get
```

## 🔐 安全加固

### Q: 如何保护 API 密钥？

**A:** 
1. 使用环境变量：
```bash
export OPENCLAW_API_KEY=xxx
```

2. 使用 1Password 集成：
```json
{
  "plugins": {
    "1password": {
      "vault": "Personal",
      "item": "OpenClaw Keys"
    }
  }
}
```

3. 不要提交配置文件到 Git

### Q: 如何限制访问？

**A:** 配置访问控制：
```json
{
  "security": {
    "allowlist": ["+86138****", "@trusted_user"],
    "rateLimit": {
      "requests": 100,
      "window": 3600
    }
  }
}
```

### Q: 如何审计操作日志？

**A:** 
```bash
# 查看完整日志
openclaw logs --level debug

# 搜索特定操作
openclaw logs --grep "message send"

# 导出日志
openclaw logs --output /path/to/log.txt
```

## 🌐 多通道

### Q: 如何配置多个 Telegram Bot？

**A:** 
```json
{
  "plugins": {
    "telegram": [
      { "token": "bot1_token", "name": "Personal" },
      { "token": "bot2_token", "name": "Work" }
    ]
  }
}
```

### Q: WhatsApp 连接失败？

**A:** 
1. 检查 WhatsApp Business API 配置
2. 验证 Webhook URL
3. 确认电话号码格式（+86...）
4. 查看连接日志

### Q: 如何区分不同渠道的消息？

**A:** 在技能中检查 `context.channel`：

```javascript
if (context.channel === 'telegram') {
  // Telegram 特定逻辑
} else if (context.channel === 'whatsapp') {
  // WhatsApp 特定逻辑
}
```

## 💾 数据管理

### Q: 如何备份配置？

**A:** 
```bash
# 备份整个配置目录
cp -r ~/.openclaw ~/.openclaw.backup

# 或只备份配置文件
cp ~/.openclaw/config.json ~/backups/config-$(date +%Y%m%d).json
```

### Q: 如何迁移到新设备？

**A:** 
1. 备份 `~/.openclaw/`
2. 在新设备安装 OpenClaw
3. 恢复备份
4. 重新授权 OAuth（飞书、Google 等）
5. 重启网关

### Q: 会话历史保存在哪？

**A:** 
```
~/.openclaw/agents/main/sessions/
└── <session-id>.jsonl
```

查看历史：
```bash
cat ~/.openclaw/agents/main/sessions/<session-id>.jsonl
```

## 🐛 故障排查

### Q: 网关启动失败？

**A:** 
```bash
# 查看详细错误
openclaw gateway start --verbose

# 检查端口占用
lsof -i :PORT

# 查看系统日志
journalctl -u openclaw
```

### Q: 插件加载失败？

**A:** 
```bash
# 列出已加载插件
openclaw plugins list

# 查看插件日志
openclaw logs --grep plugin

# 重新安装插件
npm install -g @openclaw/plugin-name
```

### Q: 如何重置配置？

**A:** 
```bash
# 备份当前配置
mv ~/.openclaw/config.json ~/.openclaw/config.json.backup

# 重新初始化
openclaw onboard

# 或手动创建最小配置
cat > ~/.openclaw/config.json <<EOF
{
  "plugins": {
    "entries": []
  }
}
EOF
```
