# FAQ 10：升级与迁移

> 版本升级、数据迁移、配置变更的完整操作指南。

---

## Q1：如何查看当前版本并检查是否有新版本？

```bash
# 查看当前版本
openclaw --version

# 检查最新版本
openclaw update check

# 或直接查看 GitHub Releases
curl -s https://api.github.com/repos/openclaw/openclaw/releases/latest | jq .tag_name
```

---

## Q2：标准升级流程是什么？

### 升级前准备（必做）

```bash
# 1. 备份数据（最重要！）
openclaw backup create --label "pre-upgrade-$(openclaw --version)"

# 2. 查看升级说明
openclaw update notes v2.x.x

# 3. 在测试环境先升级验证
```

### 执行升级

```bash
# 方式 1：npm 升级
npm update -g @openclaw/cli
openclaw --version  # 确认版本

# 方式 2：Docker 升级
docker pull openclaw/openclaw:latest
docker compose down
docker compose up -d

# 方式 3：从源码升级
git pull origin main
npm install
npm run build
pm2 restart openclaw
```

### 升级后验证

```bash
# 执行数据库迁移（如有）
openclaw db migrate

# 健康检查
openclaw health check

# 功能验证
openclaw test --smoke
```

## Q3：如何处理破坏性变更（Breaking Changes）？

查看 `CHANGELOG.md` 中标注 `⚠️ BREAKING` 的条目：

```bash
# 查看变更历史
openclaw changelog --breaking-only

# 示例破坏性变更处理：
# v2.0 将配置项 `ai.model` 改为 `ai.default_model`
```

```yaml
# ❌ v1.x 旧配置
ai:
  model: gpt-4

# ✅ v2.x 新配置
ai:
  default_model: gpt-4
  fallback_model: gpt-3.5-turbo
```

```bash
# 使用迁移工具自动转换配置
openclaw config migrate --from v1 --to v2
```

## Q4：如何从 SQLite 迁移到 PostgreSQL？

适合从个人部署升级到团队部署场景：

```bash
#!/bin/bash
# 一键迁移脚本

# 1. 导出 SQLite 数据
openclaw db export --format sql --output /tmp/openclaw_export.sql

# 2. 创建 PostgreSQL 数据库
createdb openclaw_prod

# 3. 导入到 PostgreSQL
psql openclaw_prod < /tmp/openclaw_export.sql

# 4. 更新配置
cat >> openclaw.config.yaml << EOF
database:
  type: postgresql
  url: postgresql://user:password@localhost:5432/openclaw_prod
EOF

# 5. 验证
openclaw db status
```

## Q5：升级失败如何回滚？

```bash
# 回滚到上一个版本
openclaw rollback

# 或指定版本
npm install -g @openclaw/cli@1.9.0

# 回滚数据库迁移
openclaw db migrate:rollback --steps 1

# 从备份恢复（最彻底的方式）
openclaw backup restore --label "pre-upgrade-v1.9.0"
```

## Q7：配置文件格式变更了怎么办？

```bash
# 验证当前配置是否兼容新版本
openclaw config validate --version v2

# 自动迁移（有备份的前提下）
openclaw config migrate --backup --in-place

# 查看具体差异
openclaw config diff --current --target v2
```

## Q8：插件升级与兼容性问题？

```bash
# 查看插件兼容性
openclaw plugins list --show-compatibility

# 升级所有插件
openclaw plugins update --all

# 升级指定插件
openclaw plugins update my-plugin@2.0.0

# 禁用不兼容插件
openclaw plugins disable incompatible-plugin
```
